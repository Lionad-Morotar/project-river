## Affected Pages/Routes
- `/` — 主页，展示 Git 历史时间河流（Streamgraph）和月份选择器
- `/embed/:projectId` — iframe 嵌入路由，展示简化版交互图表
- API `/api/projects/:id/monthly` — 返回月度聚合数据
- API `/api/projects/:id/contributors/:email` — 返回贡献者详情

## Key Interactions to Verify
- 点击河流图（Streamgraph）中某个月份的垂直切片，右侧/下方详情面板更新为该月数据
- hover/点击河流图中的某一层彩带，高亮该 contributor 的时间轨迹并可选进入贡献者详情视图
- 拖拽/点击月份选择条（Month Brush），高亮对应时间范围
- Top Contributors 列表正确显示头像和贡献数
- 24h 极坐标图在 hover 时显示具体小时和提交数

## Edge Cases
- 空仓库（0 commits）
- 仓库只有 1 个 commit
- 仓库有 100k+ commits
- 贡献者使用不同 email（同一人多邮箱去重逻辑）
- merge commit 的 parents 解析
- 无 tag 的仓库
- 网络超时导致 GitHub Avatar API 失败时的 Gravatar 回退
- 浏览器缩放/窗口大小变化时 D3 图表重绘

## Critical Paths
- Git 解析 → 数据计算 → 数据库写入 → API 查询 → 前端渲染 的完整 E2E 流程
- SVG/PNG 导出功能生成有效文件
