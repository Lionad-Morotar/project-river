## Locked decisions to implement

- **D-01:** Route lives at `apps/web/server/api/projects/[id]/daily.get.ts`.
- **D-02:** Reads database via `DATABASE_URL` from root `.env` using `@project-river/db/client`.
- **D-03:** Returns long format rows with fields: `date`, `contributor`, `commits`, `linesAdded`, `linesDeleted`, `filesTouched`, `cumulativeCommits`.
- **D-04:** Supports optional `?start=YYYY-MM-DD` and `?end=YYYY-MM-DD`.
- **D-05:** No range provided = return entire project history.
- **D-06:** Backend uses PostgreSQL `generate_series` to ensure continuous date range and fills missing contributor-days with `0`.
- **D-08:** Daily endpoint joins with `sum_day` to include `cumulativeCommits`.
- **D-10:** Results sorted by `date` ASC, then `contributor` ASC.
- **D-12:** Nonexistent `projectId` returns `404` with JSON error.
- **D-13:** Malformed `start`/`end` parameters return `400`.
- **D-14:** Supports `?limit=N` and `?offset=N`. Default `limit=1000`, maximum `5000`.

## Research-verified SQL skeleton

```typescript
import { z } from 'zod'
import { sql } from 'drizzle-orm'
import { db } from '@project-river/db/client'

const querySchema = z.object({
  start: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  end: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  limit: z.coerce.number().int().min(1).max(5000).default(1000),
  offset: z.coerce.number().int().min(0).default(0),
})

export default defineEventHandler(async (event) => {
  const projectId = Number(getRouterParam(event, 'id'))
  if (!Number.isFinite(projectId)) {
    throw createError({ statusCode: 400, statusMessage: 'Invalid project ID' })
  }

  const { start, end, limit, offset } = await getValidatedQuery(event, querySchema.parse)

  const projectCheck = await db.execute(sql`SELECT 1 FROM projects WHERE id = ${projectId} LIMIT 1`)
  if (projectCheck.rowCount === 0) {
    throw createError({ statusCode: 404, statusMessage: 'Project not found' })
  }

  const result = await db.execute(sql`
    WITH bounds AS (
      SELECT
        COALESCE(${start ? sql`${start}::date` : sql`(SELECT MIN(date) FROM daily_stats WHERE project_id = ${projectId})`}, CURRENT_DATE) AS min_date,
        COALESCE(${end ? sql`${end}::date` : sql`(SELECT MAX(date) FROM daily_stats WHERE project_id = ${projectId})`}, CURRENT_DATE) AS max_date
    ),
    contributors AS (
      SELECT DISTINCT contributor FROM daily_stats WHERE project_id = ${projectId}
    ),
    date_range AS (
      SELECT generate_series(min_date, max_date, '1 day'::interval)::date AS date
      FROM bounds
    ),
    grid AS (
      SELECT d.date, c.contributor
      FROM date_range d
      CROSS JOIN contributors c
    )
    SELECT
      g.date AS "date",
      g.contributor AS "contributor",
      COALESCE(ds.commits, 0) AS "commits",
      COALESCE(ds.insertions, 0) AS "linesAdded",
      COALESCE(ds.deletions, 0) AS "linesDeleted",
      COALESCE(ds.files_touched, 0) AS "filesTouched",
      COALESCE(sd.cumulative_commits, 0) AS "cumulativeCommits"
    FROM grid g
    LEFT JOIN daily_stats ds
      ON ds.project_id = ${projectId}
     AND ds.date = g.date
     AND ds.contributor = g.contributor
    LEFT JOIN sum_day sd
      ON sd.project_id = ${projectId}
     AND sd.date = g.date
     AND sd.contributor = g.contributor
    ORDER BY g.date ASC, g.contributor ASC
    LIMIT ${limit} OFFSET ${offset}
  `)

  return result.rows.map((r: any) => ({
    date: r.date,
    contributor: r.contributor,
    commits: Number(r.commits),
    linesAdded: Number(r.linesAdded),
    linesDeleted: Number(r.linesDeleted),
    filesTouched: Number(r.filesTouched),
    cumulativeCommits: Number(r.cumulativeCommits),
  }))
})
```

## Known issues to avoid

From RESEARCH.md:
- Pitfall 1: If a project has no `daily_stats`, `CROSS JOIN contributors` yields 0 rows. The pre-query existence check handles the 404 case; returning an empty array for an existing project with no stats is acceptable.
- Pitfall 2: Use double-quoted aliases in SQL (`AS "cumulativeCommits"`) and map rows in TypeScript before returning.
- Pitfall 6: Always write `throw createError({ ... })` — do not omit `throw`.
