# apps/web Code Review

**Plan:** 07.1-03  
**Scope:** `apps/web` frontend components, API routes, utilities, and tests.  
**Date:** 2026-04-09

---

## Summary

`apps/web` implements a Nuxt v4 SPA (`ssr: false`) that renders an interactive D3 Streamgraph with month selection, hover tooltips, a draggable detail panel, and SVG export. The implementation follows Vue 3 Composition API patterns and uses Tailwind CSS v4 via the CSS-first `@import` approach. Server API routes (`daily.get.ts`, `monthly.get.ts`) provide time-series data using parameterized Drizzle `sql` queries with zero-fill via CTEs.

Overall, the codebase is well-structured for a Phase 1 MVP, with explicit H3 imports for testability and dedicated utilities for D3 data preparation. However, there are critical test failures when `DATABASE_URL` is absent, non-critical type-safety gaps in D3-related `any` usage, and minor build sourcemap warnings from the Tailwind/vite plugins.

## Code Quality

### Vue / Nuxt Patterns

- **Streamgraph.vue** (lines 1–313): Uses `<script setup>` cleanly, defines explicit `Props` and `emit` types, and exposes `getSvg` via `defineExpose`. The component destroys and re-creates the entire SVG on every resize/data change, which is simple but not the most performant approach for large datasets.
- **MonthDetailPanel.vue** (lines 1–226): Uses `@vueuse/core` `useDraggable` and `useWindowSize` for a floating, dockable panel. The drag handle is only rendered when `dockedEdge === 'right'`, which means users cannot drag from the left/bottom edges without first re-docking to the right. Empty state is handled with a `v-else` block.
- **MonthSelector.vue** (lines 1–28): Thin wrapper around `USelectMenu`; computed `items` correctly injects an `'All history'` null option.
- **pages/projects/[id]/index.vue** (lines 1–194): Fetches `daily` and `monthly` in parallel with `Promise.all`. Defaults `selectedMonth` to the most recent available month. Chart dimensions are driven by a window resize listener (not `ResizeObserver`), consistent with the Phase 6 decision for SSR safety.

### Tailwind / Styling

- Tailwind utility classes are used consistently.
- `bg-slate-900` graph container and `bg-slate-800` panel maintain a coherent dark theme.
- Tooltip uses `absolute` positioning offset by `+12px`; there is no viewport-boundary clamping, so tooltips near the right/bottom edges can overflow the viewport.

### Accessibility

- **MonthDetailPanel.vue** (line 122, 137, 147) includes `aria-label` on navigation buttons, but the drag handle only has `aria-label="Drag panel"` when docked right.
- **StreamgraphTooltip.vue** is conditionally rendered with `v-if="visible"`; no live-region or `role="tooltip"` is present, so screen-reader users may miss hover content.

## Type Safety

### Well-typed areas

- **API route validation** (`daily.get.ts` / `monthly.get.ts`, lines 6–11): Uses Zod with `z.coerce.number()` for `limit`/`offset`, and regex-validated date strings.
- **Props interfaces** in all Vue components are explicit.
- **Utilities** (`d3Helpers.ts`, `monthDetailHelpers.ts`, `svgExport.ts`) define and export TypeScript interfaces (`DailyRow`, `PivotedRow`, `MonthContributor`, `MonthlyRow`).

### Gaps

- **Streamgraph.vue** uses many `any` types for D3 selections and behaviors (lines 64–76, 120–124, etc.). Example:
  ```ts
  let gChart: any = null
  let areaGenerator: any = null
  ```
  These should be tightened to `Selection<..., ..., ..., ...>` and `Area<...>` types for compile-time safety.
- **Event typing in D3 callbacks** (lines 163, 193, 212): The zoom/brush event handlers are typed as `event: any`, losing type information for `event.transform` and `event.selection`.
- **API route data mapping** (lines 65–73 in `daily.get.ts`, lines 61–68 in `monthly.get.ts`): Casts rows to `any` before mapping; a narrow interface for the raw SQL result would be preferable.

## D3 / Streamgraph Integration

### Stack Construction

- **d3Helpers.ts** (lines 51–56):
  ```ts
  export function buildStack(contributors: string[], data: PivotedRow[]) {
    return d3Stack()
      .keys(contributors)
      .order(stackOrderInsideOut)
      .offset(stackOffsetWiggle)(data)
  }
  ```
  This matches the Phase 6 UI spec for a wiggle-centered Streamgraph.

### Zoom / Brush

- **Streamgraph.vue** (lines 189–207, 209–220):
  - `scaleExtent([1, 50])` allows deep zoom.
  - The zoom `extent` and `translateExtent` are correctly clamped to the chart horizontal bounds.
  - Mutual update between zoom and brush is guarded with `event.sourceEvent` type checks, preventing infinite loops.

### Tooltip Positioning

- **Streamgraph.vue** (lines 161–187): Pointer events are bound to individual layer paths. `d3.pointer(event, svg.node())` is used to derive the hovered date via `xScale.invert(px)`.
- **Issue:** The fallback search (`Math.abs(new Date(r.date).getTime() - date.getTime()) <= 86400000`) can match the wrong day if the dataset contains multiple adjacent days, because it only checks within a 24-hour window rather than snapping to the nearest day.

### Month Highlight Logic

- **Streamgraph.vue** (lines 251–278): `updateMonthHighlight` computes start/end of the selected month, maps them through `xScale`, and clamps the rectangle to the visible chart area (`marginLeft` … `marginLeft + chartWidth`). This satisfies the Phase 6 spec to avoid bleed outside the clip path.

### SVG Export Serialization

- **svgExport.ts** (lines 1–75): `serializeSvgWithLegend` clones the SVG, injects an inline `<style>` block, and appends a legend group. It limits the legend to the top 10 contributors with a "+N more" fallback. `downloadStreamgraphSvg` constructs a Blob and triggers a download via an ephemeral `<a>` element, revoking the object URL afterward.
- **Minor issue:** The legend background uses `rgba(15,23,42,0.85)`, which may render differently in viewers that do not support transparency (some SVG importers).

## API Route Design

### Validation

- Both routes validate `projectId` via `Number(...)` and `Number.isFinite` (lines 14–17), rejecting non-numeric IDs with 400.
- Query schemas enforce ISO date format (`YYYY-MM-DD`) and bounded `limit`/`offset`.

### Zero-Filling

- **daily.get.ts** uses a CTE grid (`date_range`, `grid`) that cross-joins distinct contributors with the date range, ensuring every contributor has a row for every day in the bounds.
- **monthly.get.ts** uses a similar `month_range` / `grid` CTE for month-level zero-filling.

### Error Handling

- 404 is returned when `projectCheck.rowCount === 0` (lines 21–24).
- **Issue:** When `DATABASE_URL` is absent, the project existence query throws a `DrizzleQueryError` (SASL authentication failure) before the 404 logic can execute. This causes two tests (`daily.get.test.ts` line 51 and `monthly.get.test.ts` line 49) to fail because they expect a 404 but receive a raw database error. The route handlers do not catch DB connection errors and map them to HTTP errors.

### SQL Injection Prevention

- All user-facing parameters are passed via Drizzle's tagged `sql` template literals (e.g., `sql\`... WHERE id = ${projectId}\``), which use parameterized queries. This is safe.
- **Caveat:** Dynamic `start`/`end` injection uses a ternary with `sql\`${start}::date\``, but `start` is Zod-validated to match `/^\d{4}-\d{2}-\d{2}$/` before being interpolated, so the attack surface is minimal.

## Findings

### Critical

| # | Finding | File | Line(s) | Impact |
|---|---------|------|---------|--------|
| C1 | Database connection errors surface as raw `DrizzleQueryError` instead of 500/503 | `daily.get.ts`, `monthly.get.ts` | 21–24 | Tests fail when `DATABASE_URL` is missing; leaks internal error details |
| C2 | Tooltip date resolution fallback can match an adjacent incorrect day | `Streamgraph.vue` | 171 | Users may see stats for the wrong date near layer boundaries |

### Non-Critical

| # | Finding | File | Line(s) | Impact |
|---|---------|------|---------|--------|
| N1 | Extensive `any` typing in D3 selections/generators | `Streamgraph.vue` | 64–76, 120–124, 163, 193, 212 | Reduced type safety, harder refactors |
| N2 | Tooltip has no viewport boundary clamping | `StreamgraphTooltip.vue` | 21 | Tooltip may render partially off-screen |
| N3 | Drag handle absent for left/bottom docking edges | `MonthDetailPanel.vue` | 119–127 | Panel cannot be dragged once snapped to bottom/left |
| N4 | Build sourcemap warnings from Tailwind/vite plugins | Build output | — | Noise in build logs; no functional impact |
| N5 | API raw row mapping uses `any` cast | `daily.get.ts`, `monthly.get.ts` | 65+, 61+ | Could be tightened with an explicit SQL result interface |
| N6 | streamgraphRef type duplication: `{ getSvg: () => SVGSVGElement \| null } \| null` | `pages/projects/[id]/index.vue` | 36 | Should ideally import the component's exposed type rather than duplicating the shape |

---

## Recommendations

1. **Wrap DB calls in try/catch** in API routes to convert connection/authentication errors into `createError({ statusCode: 500 })` so 404 tests are deterministic regardless of database availability.
2. **Improve tooltip date snapping:** Use `d3.timeDay.round(date)` or find the nearest date entry rather than a ±24-hour tolerance.
3. **Add viewport clamping to tooltip** (e.g., compute against `window.innerWidth/Height`).
4. **Replace D3 `any` annotations** with `@types/d3` generics (`Selection`, `Area`, `ZoomBehavior`, `BrushBehavior`).
5. **Add drag handles for left/bottom edges** or refactor docking logic so the handle is always present on the outer edge of the panel.
6. **Define a `SqlDailyRow` / `SqlMonthlyRow` interface** for the raw query result mapping to eliminate `any` casts in route handlers.
